# Code of Conduct

This project follows a professional, technical collaboration standard. Be
specific, respectful, and focused on improving the software.

## Expected behavior

- Discuss code, contracts, tests, and evidence rather than personal intent.
- Make compatibility, safety, and migration risks explicit.
- Respect maintainer decisions on release scope and supported surfaces.
- Report security issues privately through the process in `SECURITY.md`.

## Unacceptable behavior

- Harassment, discrimination, threats, or sustained personal attacks.
- Publishing private security reports before coordinated disclosure.
- Pressuring maintainers to weaken leakage, ABI, or security invariants for
  convenience.

Maintainers may edit, close, lock, or remove contributions that violate this
standard.
